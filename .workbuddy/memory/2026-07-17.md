## Mineradio iPad 后台音频播放方案 — 原生 AVPlayer 单一播放器

### 问题
iOS WKWebView 中 HTML5 `<audio>` 进入后台被系统强制暂停，`UIBackgroundModes:audio` 对 WKWebView 不生效。
原"双播放器"方案（网页 audio 出声 + 原生 bgPlayer 静音跟随/后台切原生）有同步断裂、重复下载问题。

### 解决方案：原生 AVPlayer 接管 + MTAudioProcessingTap 频谱回传
- 网页端用 `NativeAudioAdapter`（伪装 HTMLAudioElement）把 play/pause/seek/src 转发到原生 AVPlayer
- AVPlayer 是唯一出声源，天然支持后台播放
- `MTAudioProcessingTap` 窃听 AVPlayer PCM 采样 → `vDSP FFT`(Hann 窗, 2048→1024 频段) → base64 推回网页
- 网页 `analyser.getByteFrequencyData()` 读原生频谱，three.js 可视化器无需改动
- `os_unfair_lock trylock` 保护环形缓冲，音频线程不阻塞
- `CADisplayLink` 30fps 推送频谱+时间，`evaluateJavaScript` 单次调用携带全部数据
- 锁屏控制 `MPRemoteCommandCenter` + `MPNowPlayingInfoCenter`

### 关键发现
- 网页端用 Web Audio API `createMediaElementSource(audio)` 喂 analyser 做可视化——纯原生播放后无源可接，必须用 MTAudioProcessingTap 补上频谱通路
- `initAudio()` 在原生模式跳过 `createMediaElementSource`，改用 `installNativeAnalyserShim` 覆盖 `getByteFrequencyData`
- 音量/淡入淡出函数（`applyVolumeToAudio`/`rampAudioOutputGain`/`setAudioOutputGainImmediate`）需原生分支转发到 `audio.volume`→原生 `setVolume`
- `togglePipWindow` 原生模式交给原生 PiPManager，跳过浏览器 canvas 画中画（含 `createMediaElementSource`）

### Xcode 26.6 / iOS 26.5 SDK API 变更修复
- MTAudioProcessingTapRef 已重命名为 MTAudioProcessingTap
- MTAudioProcessingTapCreate 新增第4参数 flags (传 0)
- MTAudioProcessingTapRelease 改为 CFRelease → 实际 ARC 管理 (直接置 nil)
- vDSP FFT 旧 C API (vDSP_create_fftsetup 等) 已废弃
- 新 vDSP.DFT 初始化: vDSP.DFT(count:direction:transformType:ofType:)
- DFTTransformType 只有 .complexComplex 和 .complexReal (没有 .realReal)
- WindowSequence 改名为 .hanningNormalized/.hanningDenormalized (不是 .hann)
- vDSP.multiply/.absolute/.convert 在新 SDK 中存在重载歧义，改用手动循环

### 最终 FFT 方案
- vDSP.DFT(count: 1024, direction: .forward, transformType: .complexComplex, ofType: Float.self)
- dft.transform(inputReal:inputImaginary:) -> (real, imaginary) 元组
- 手动计算 magnitude, dB, 归一化 (避开 vDSP 辅助函数重载歧义)
